This JSON object represents the data payload that the frontend will send when a new issue is reported.

Endpoint:       POST /api/issues (or whatever your agreed-upon endpoint is)
Content-Type:   application/json

Field Explanations:

title      (string): The brief title of the issue. (Required)
location   (string): Where the issue occurred (e.g., "Room 101", "Lobby"). (Required)
description(string): Detailed explanation of the issue. (Required)
priority   (string): The urgency of the issue. Expected values: "Low", "Medium", "High". (Required)
status     (string): The initial status of the issue. The frontend currently sends "pending". (Required by frontend, backend can validate/override).
createdAt  (string): ISO 8601 formatted timestamp of when the issue was reported (e.g., "2024-06-19T16:30:00.000Z"). This is generated by the frontend. (Required)
